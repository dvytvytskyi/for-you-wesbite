# 📋 Технічне завдання: Виправлення фільтрів Property Finder API

**Об'єкт**: Ендпоінт `/api/property-finder/projects`

На даний момент фронтенд передає всі необхідні параметри в URL, але бекенд повертає повний список об'єктів без врахування фільтрації. Необхідно реалізувати обробку наступних параметрів на стороні API.

## 1. Фільтр по спальнях (Bedrooms)
- **Параметр**: `bedrooms`
- **Очікувана поведінка**:
  - Приймати кілька значень (наприклад, `?bedrooms=1&bedrooms=2` або `?bedrooms=1,2`).
  - Повертати об'єкти, де кількість спалень відповідає будь-якому з обраних значень.
  - Для значень `6+` — повертати об'єкти, де кількість спалень >= 6.
  - Поле в БД: `bedrooms` у таблиці `property_finder_projects` або відповідне поле у `fullData`.

## 2. Фільтр за ціною (Price)
- **Параметри**: `priceMin`, `priceMax`
- **Очікувана поведінка**:
  - Фільтрувати об'єкти за полем ціни (зазвичай `startingPrice` або `min_price`).
  - Значення передаються в **AED**.
  - Якщо `priceMin` — ціна >= значення.
  - Якщо `priceMax` — ціна <= значення.

## 3. Фільтр за площею (Size)
- **Параметри**: `sizeMin`, `sizeMax`
- **Очікувана поведінка**:
  - Фільтрувати за площею (поле `area_size` або `size`).
  - Значення передаються в **sq. ft.** (квадратні фути).

## 4. Фільтр меблювання (Furnishing)
- **Параметр**: `furnishingType`
- **Можливі значення**: `furnished`, `unfurnished`, `partly-furnished`.
- **Поле в БД**: `furnishingType`.

## 5. Фільтр по девелоперу (Developer)
- **Параметр**: `developerId`
- **Очікувана поведінка**:
  - Фільтрація об'єктів за ID розробника.

## 6. Пошук (Search)
- **Параметр**: `search`
- **Очікувана поведінка**:
  - Регістронезалежний пошук (LIKE) по назві проекту (`name`), розробника та району.

## 7. Фільтр за типом оголошення (Listing Type)
- **Параметр**: `type`
- **Очікувана поведінка**:
  - Фільтрація за типом: `sale` (продаж) або `rent` (оренда).
  - У випадку оренди, фільтри за ціною (Price) мають працювати з ціною оренди (на місяць/рік).
- **Поле в БД**: `listing_type` або `offering_type`.

## 8. Список локацій (Locations List)
- **Об'єкт**: НОВИЙ Ендпоінт `/api/property-finder/locations`
- **Очікувана поведінка**:
  - Повертати список унікальних назв районів/локацій, які присутні в базі даних проектів (диференціал по полю `location`).
  - Список має бути відсортований за алфавітом.
  - Кожен елемент має містити `id` (якщо є) або просто назву для фільтрації.
- **Використання**: Фронтенд використовуватиме цей список для заповнення випадаючого списку (Dropdown).

---

### Приклад повного запиту з фронтенду:
`GET /api/property-finder/projects?type=sale&status=off_plan&bedrooms=2&bedrooms=3&priceMin=1000000&priceMax=5000000&furnishingType=furnished&page=1&limit=24`

**Важливо**: Після реалізації фільтрації, параметр `total` у відповіді API повинен повертати кількість об'єктів, що відповідають фільтрам, а не загальну кількість у базі.
